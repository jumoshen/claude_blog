# T1: 文章分类管理

## 需求分析
文章分类管理是内容增强的基础功能，允许博主对文章进行归类整理。

## 实现方案

### 1. 数据库设计
```sql
CREATE TABLE IF NOT EXISTS `categories` (
  `id` BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `name` VARCHAR(100) NOT NULL COMMENT '分类名称',
  `slug` VARCHAR(100) NOT NULL COMMENT '分类别名(URL友好)',
  `description` VARCHAR(500) DEFAULT '' COMMENT '分类描述',
  `sort` INT DEFAULT 0 COMMENT '排序权重',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `deleted_at` DATETIME DEFAULT NULL COMMENT '软删除',
  UNIQUE KEY `uk_slug` (`slug`),
  KEY `idx_sort` (`sort`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文章分类表';

-- 文章表新增分类字段
ALTER TABLE `articles` ADD COLUMN `category_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '分类ID' AFTER `tags`;
```

### 2. API 设计

| 方法 | 路径 | 说明 |
|-----|------|------|
| GET | /api/admin/categories | 分类列表(分页) |
| GET | /api/admin/categories/all | 全部分类(下拉框用) |
| GET | /api/admin/categories/:id | 分类详情 |
| POST | /api/admin/categories | 创建分类 |
| PUT | /api/admin/categories/:id | 更新分类 |
| DELETE | /api/admin/categories/:id | 删除分类 |

### 3. 业务逻辑
- 创建/更新分类时，slug 自动从 name 转换（中文转拼音或英文）
- 删除分类时，若有文章使用该分类，提示无法删除
- 分类支持排序

### 4. 前端(B端)
- 分类管理页面：列表、新增、编辑、删除
- 文章编辑页：分类下拉选择
