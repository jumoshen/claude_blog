# T2: 文章标签管理

## 需求分析
文章标签管理允许博主为文章添加多个标签，实现文章归类。

## 实现方案

### 1. 数据库设计
```sql
CREATE TABLE IF NOT EXISTS `tags` (
  `id` BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `name` VARCHAR(100) NOT NULL COMMENT '标签名称',
  `slug` VARCHAR(100) NOT NULL COMMENT '标签别名(URL友好)',
  `color` VARCHAR(20) DEFAULT '' COMMENT '标签颜色',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `deleted_at` DATETIME DEFAULT NULL COMMENT '软删除',
  UNIQUE KEY `uk_slug` (`slug`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文章标签表';

-- 文章标签关联表
CREATE TABLE IF NOT EXISTS `post_tags` (
  `id` BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `post_id` BIGINT UNSIGNED NOT NULL COMMENT '文章ID',
  `tag_id` BIGINT UNSIGNED NOT NULL COMMENT '标签ID',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_post_tag` (`post_id`, `tag_id`),
  KEY `idx_tag_id` (`tag_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文章标签关联表';
```

### 2. API 设计

| 方法 | 路径 | 说明 |
|-----|------|------|
| GET | /api/admin/tags | 标签列表(分页) |
| GET | /api/admin/tags/all | 全部标签(下拉框用) |
| GET | /api/admin/tags/:id | 标签详情 |
| POST | /api/admin/tags | 创建标签 |
| PUT | /api/admin/tags/:id | 更新标签 |
| DELETE | /api/admin/tags/:id | 删除标签 |

### 3. 业务逻辑
- 标签支持颜色自定义
- 多对多关联通过 post_tags 表实现
- 删除标签时自动清理关联
